# 내 의원·내 예산 — 시민 정치 책무성 플랫폼

> *"내 한 표가 어디로 갔는지, 내 세금이 어떻게 쓰이는지 시민이 한 화면에서 본다."*

## 한 줄 요약

대한민국 시민이 본인 지역의 **국회의원·광역의원·기초의원**, **2026.6.3 지방선거 후보**, **지자체 예산·결산**, **법안·조례안·회의록**을 한 페이지에서 확인하고 책무를 묻을 수 있게 하는 **오픈 데이터 기반 정치 투명성 플랫폼**.

---

## 프로젝트 목표

### 핵심 미션
1. **정보 비대칭 해소** — 정부·국회·NEC가 공개하는 데이터를 시민이 이해하기 쉬운 형태로 통합
2. **책무성 강화** — "내 표를 받은 사람이 무엇을 했는가"를 가시화 (발의·표결·발언·예산 집행)
3. **선택의 근거 제공** — 지방선거·총선 등 투표 전 후보 정보·공약·이행률 비교
4. **중립성 유지** — LLM이 단독으로 정치적 판단을 내리지 않음. 모든 평가는 **공식 출처·통계 지표**에 기반

### 누구를 위한 서비스인가
- **본인 지역구가 누구인지 모르는 시민** — 주소만 입력하면 자기 의원/후보 즉시 표시
- **선거를 앞두고 후보 비교가 필요한 유권자** — 공약·경력·재산·전과를 한 화면에서
- **지방재정에 관심 있는 시민** — 우리 동네 결산을 분야별로 클릭해 들여다보기
- **언론·연구자·시민단체** — 책무성 자료의 출발점

## 주요 기능

### 1. 의원 정보 (NATIONAL · PROVINCIAL · BASIC)
- 국회의원 286명 + 광역의원 872명 + 기초의원 3,563명
- 발의 법안·표결 이력 (NATIONAL)
- 조례안·건의안·회의록 (PROVINCIAL·BASIC, CLIK 연동)
- 신고 재산·전과·병역·세금 (공직자윤리위원회)
- 사진 자동 수집 (CLIK assemblyinfo API)
- **"주요 뉴스" + "논란" 탭** — Google News RSS + 휴리스틱 분류
- **AI 법안 요약** (Gemini 그라운딩) — 제안이유·주요내용·개정 변경점

### 2. 지역 허브 (`/region-hub`)
- 주소 한 줄 입력하면 본인 시·도/시·군·구의 모든 정치 정보 통합
- 우리 지역 의원 카드 + 예산 결산 + 후보 + 외부 링크
- "내 지역" localStorage에 저장 → 후보 페이지·뉴스 자동 필터

### 3. 예산·결산 (`/budget`)
- 국가 예산 (열린재정포털)
- 광역·기초 결산 (지방재정365)
- 분야별 클릭해서 부문별 세부사항 우측 패널 표시
- 결산서 PDF 직접 링크

### 4. 후보 정보 (`/candidates`) — 2026.6.3 지방선거
- 시·도지사 104명 + 시·군·구청장 1,389명 (현재 적재 완료)
- 광역의원·기초의원·교육감·비례 (코드 준비 완료, NEC 데이터 대기)
- 이름·지역 검색 + 직위·내 지역 자동 필터

### 5. 회의록 뷰어 (`/minutes/[docId]`)
- CLIK 회의록 원문을 **카카오톡 채팅 스타일** 으로 렌더
- 의장은 우측(노란 톤), 의원은 좌측(이름 해시 기반 컬러)
- 의원 사진 아바타 자동 노출
- **Gemini AI 분석** — 전체 요약 + 발언자별 요약 + 핵심 주제
- 장문 발언은 자동 단락 분리 + "더 보기" 토글

### 6. 조례안·건의안 AI 요약
- CLIK API는 메타데이터만 제공 → Gemini 그라운딩으로 본문 검색·요약
- 의안 카드의 "내용 보기" 토글 → AI 요약 + 개정 변경점 + 출처

## 핵심 설계 원칙

### 1. 중립성
- LLM이 "거짓이다·잘못이다"라고 단언하지 않음
- AI 요약에 항상 "원문 직접 확인 권장" 안내
- 뉴스 분류는 **출처 시그널 집계** (다양성·1차 출처·정정보도 비율)이지 의견 평가가 아님

### 2. 출처 우선
- 모든 데이터에 원본 링크 노출 (CLIK·NEC·국회·지방재정365)
- LLM 요약은 보조, 검증은 사용자 몫

### 3. 개인정보 최소화
- 회원가입·로그인 없음
- "내 지역"은 localStorage 익명 저장
- 외부 트래커 없음

### 4. 데이터 신선도
- 일일 cron: 후보 등록 변경·신고 공시 갱신
- 의원 사진·회의록은 점진적 ingest
- 캐시·재생성 사이클 명시적으로 노출

## 미해결·진행 중

### 의존성 대기
- **9회 지선 후보 공약** — NEC가 5/22~25 정식 공개 후 ingest
- **광역·기초의원·교육감 후보** — 본 등록(5/20~21) 직후 NEC 데이터 노출 대기
- **후보 사진** — NEC sggCode 매핑 brute-force 필요 (광역단체장 80% 가능 확인)

### 보류된 기능
- **"나의 표는 누구에게?"** — 후보 5대 공약 비교·평가 (공약 데이터 대기)
- **단체장 공약 이행률** — 4년 전 당선인 공약 vs 현재 (수동 매핑 필요)
- **시민단체 의정활동 평가 통합** — 참여연대 등 외부 평가 자료
- **지방의회 해외출장 보고서** — 정보공개포털 스크래핑

### 운영 과제
- 광역·기초의원 사진 cron 자동화 (Railway cron service 수동 생성 필요)
- CLIK API 일일 1,000회 한도 분산 (사진·회의록·의안 경합)
